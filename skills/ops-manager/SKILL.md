## Weekly Review Framework (Affiliate Operations)

### Monday — Review & Plan
```
□ Pull last week's EPC/revenue by domain
□ Flag any domain below 0.35 EPC for 3+ days → rotate
□ Check email campaign stats (open rate, RPE, unsubscribes)
□ Review ad spend vs. revenue (ROAS check)
□ Set top 3 priorities for the week
```

### Wednesday — Mid-Week Check
```
□ Any domains needing urgent network shuffle?
□ Email campaigns on track?
□ Any tech issues reported (forms, tracking, redirects)?
□ Ad campaigns burning budget without conversions?
```

### Friday — Wrap & Handoff
```
□ Document what moved this week (domain changes, new pages live)
□ Note anything blocked or pending
□ Update domain status tracker
□ Schedule next week's sends/campaigns
```

## SOP Templates

### New Domain Launch Checklist
```
PRE-LAUNCH
□ Domain registered + SSL active
□ Landing page built (HTML + SEO optimized)
□ Schema markup added
□ Legal disclosures in footer
□ Privacy policy + Terms pages live
□ TCPA consent on form
□ GA4 + GTM installed and tracking
□ Test form submission end-to-end
□ Mobile responsive check
□ Page speed > 85 (PageSpeed Insights)

NETWORK SETUP
□ ZeroParallel (or chosen network) form embedded
□ Test lead submitted and confirmed received
□ Postback/pixel firing correctly

GO-LIVE
□ Set status to "Testing" in domain tracker
□ Drive small traffic (100 clicks) to verify EPC
□ If EPC > 0.35 after 3 days → promote to Active / ZP
□ If EPC < 0.25 after 5 days → pause and investigate
```

### Network Rotation SOP
```
TRIGGER: EPC drops below 0.35 for 3 consecutive days

STEPS:
1. Confirm with 3-day average (not single day spike)
2. Check if issue is traffic quality or network payout
3. Move domain: ZP → DOT (or DOT → LG)
4. Log change in domain tracker with reason
5. Monitor for 5 days at new network
6. If EPC recovers above 0.48 → promote back up
7. If still below 0.25 → retire domain from rotation
```

### New Email Campaign Launch
```
□ List segment defined and pulled
□ Suppression list applied (unsubscribes, bounces, complaints)
□ Email written and reviewed (CAN-SPAM compliance check)
□ Subject line A/B test set up (2 variants)
□ Links tested — all working
□ Unsubscribe link tested — working
□ Test send to internal team (check inbox, not spam)
□ Mail-Tester.com score > 9/10
□ Schedule send: Tue–Thu, 9–11am or 6–8pm
□ Monitor first hour: open rate, unsubscribes, complaints
□ If complaint rate > 0.08% → pause immediately
```

## Priority Framework (When Everything Feels Urgent)

Score each task:
```
Revenue Impact (1-5):  How directly does this affect revenue?
Urgency (1-5):         What breaks if this isn't done today?
Effort (1-5):          How long will this take? (5=quick win)

Priority Score = Revenue × Urgency × Effort
Do highest scores first.
```
